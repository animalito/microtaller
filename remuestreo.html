Problemas de usar un conjunto de prueba

Del ejercicio anterior podemos ver que:

  • El estimador del error de prueba es muy variable pues depende de quÃ© observaciones entraron en el conjunto de entrenamiento.
  • Usamos Ãºnicamente un subconjunto de las observaciones a nuestra disposiciÃ³n. Los mÃ©todos estadÃ­sticos tienden a ser mejores con mÃ¡s observaciones y, por ende, sobreestimamos el error de prueba con respecto a si usamos todo el conjunto de datos.
  • ValidaciÃ³n cruzada busca minimizar estos dos problemas.

Leave-one-out cross-validation (LOOCV)

  1. Dividimos nuestro conjunto de datos en dos: uno de entrenamiento y uno de prueba. El segundo lo formamos de una Ãºnica observaciÃ³n.
  2. Ajustamos el modelo en los datos de entrenamiento (\(n-1\) observaciones).
  3. Predecimos \(\hat{y_i}\) de la observaciÃ³n excluida.
  4. Calculamos \(MSE_i = (y_i - \hat{y_i})^2\)
  5. Lo repetimos para las n observaciones para obtener \(MSE_1, ..., MSE_n\).
  6. El estimador del \(MSE_{prueba}\) por LOOCV es el promedio de estos n cÃ¡lculos de error. \[ CV_{(n)} = \frac{1}{n}\sum_{i=1}^n MSE_i \]

LOOCV: ventajas y desventajas

  • Ventajas
    • Tiene menos sesgo. LOOCV tiende a no sobreestimar el error de prueba tanto como cuando usamos un solo conjunto de prueba.
    • Realizarlo mÃºltiples veces siempre da el mismo resultado pues la asignaciÃ³n en prueba y validaciÃ³n no es aleatoria.
  • Desventajas
    • Dificil de implementar pues consume mucho poder de cÃ³mputo.
    • Si \(n\) es grande, y cada modelo toma mucho tiempo en entrenar el problema se multiplica.

k-fold cross-validation

  2. Tratamos a cada uno de los \(k\) subconjuntos como un conjunto de prueba en iteraciones independientes. Es decir, para cada subconjunto, ajustamos un modelo con el resto de los datos divididos en los \(k-1\) grupos y usamos las observaciones en el grupo \(k\) como prueba.
  3. Repetimos el proceso \(k\) veces (para cada grupo de observaciones).
  4. Generamos \(k\) estimadores del error de prueba \(MSE_1, ..., MSE_k\).
  5. El estimador del \(MSE_{prueba}\) por validaciÃ³n cruzada es el promedio de estos \(k\) cÃ¡lculos de error. \[ CV_{(k)} = \frac{1}{k}\sum_{i=1}^k MSE_i \] Nota como LOOCV es un caso particular de k-fold CV en donde \(k = n\)

ValidaciÃ³n cruzada: ventajas y desventajas

  • Ventajas
    • Ventaja computacional sobre LOOVC
    • MÃ©todo general que puede ser aplicado a casi cualquier mÃ©todo de aprendizaje estadÃ­stico.
  • Desventajas
    • Hay variabilidad en el estimador del error de prueba debido a como se distribuyen las observaciones en los k-grupos. Esta variabilidad es menor que la que te da un estimador donde aleatoriamente divides el conjunto de datos en prueba y entrenamiento.
    • Hay sobre o subestimaciÃ³n del error de prueba mientras cambia la flexibilidad de los modelos utilizados.

  • Se utiliza tambiÃ©n cuando es de interÃ©s conocer el menor valor del error para contrastar entre distintos mÃ©todos posibles. En este caso, no interesa tanto el valor del estimador sino encontrar en dÃ³nde se encuentra el mÃ­nimo error, es decir, la flexibilidad del modelo.
